Setting up a RethinkDB AMI

There are two steps to complete before this instance is usable. First, the SSL certificate must be accepted by your browser to enable encryption. Second, a password needs to be set to secure the instance.

Visit RethinkDB.com for more documentation on the RethinkDB AMI.

To encrypt your session, a new self-signed certificate has been generated for this instance. This certificate is perfectly safe but it cannot be automatically verified by your browser.

When you press Continue your browser will warn you that the certificate could not be verified. If you use Chrome or Firefox, follow the instructions below to add an exception.

Chrome

  • Click on Proceed Anyway

Firefox

  • Click on I Understand the Risks
  • Click on Add Exception...
  • Click on Confirm Security Exception
